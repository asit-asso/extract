Le plugin d'archivage permet de sauver les résultats de traitement d'un élément de requête dans un répertoire. Spécifier un chemin (local ou réseau) vers lequel seront archivés les fichiers.

Le répertoire d'archivage peut être local ou sur le réseau de l'entreprise. Exemples de chemin :

  • Répertoire local Windows : D:\extract_data\...
  • Répertoire local Linux : /var/extract_data\...
  • Répertoire réseau Windows Windows (UNC) : \\server\share\folder...

Le chemin d'archivage peut contenir des variables qui seront remplacées par les propriétés de la requête. Les propriétés dynamiques autorisées sont les suivantes :

  • orderLabel
  • orderGuid
  • productGuid
  • productLabel
  • startDate
  • organism
  • client

Exemple de chemin d'archivage : /var/extraction/{orderLabel}-{orderGuid}/{productGuid}/

Important :

  • Si le répertoire cible existe et contient des fichiers portant le même nom que ceux à archiver, ils seront écrasés.
  • Si l'application Extract est déployée sur un environnement Linux, l'archivage vers un répertoire réseaux Windows ne sera pas possible.
  • Si vous souhaitez archiver vers un répertoire partagé Linux, merci de vous assurer que le montage Samba a été fait au préalable.
  • L'utilisateur de Tomcat doit avoir les privilège suffisants pour écrire dans le répertoire réseau spécifié
